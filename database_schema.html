Skema Database (ERD)

Ringkasan

Database dirancang di sekitar tabel users (autentikasi) dan tabel family_members (profil). Satu Pengguna dapat mengelola beberapa Anggota Keluarga (misalnya, "Anda", "Anak", "Pasangan").

Hubungan Utama

  1. User -> FamilyMember: One-to-Many. Satu pengguna login dan dapat beralih di antara profil keluarga yang berbeda (misalnya, mengelola diet anak mereka).
  2. FamilyMember -> FoodLog: One-to-Many. Setiap makanan terikat pada profil anggota keluarga tertentu, bukan hanya akun pengguna.
  3. FamilyMember -> GrowthLog: One-to-Many. Melacak perubahan berat/tinggi badan dari waktu ke waktu untuk grafik.
